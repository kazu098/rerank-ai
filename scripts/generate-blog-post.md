# ブログ記事生成スクリプト

このスクリプトは、プロンプトテンプレートを使用してブログ記事を生成するためのガイドです。

## 使用方法

### 1. 記事の種類とテーマを決定

```bash
# 例: Ahrefs比較記事を生成する場合
ARTICLE_TYPE="comparison"
ARTICLE_THEME="Ahrefs vs ReRank AI"
TARGET_AUDIENCE="Ahrefsを使用しているSEO担当者"
```

### 2. プロンプトテンプレートを選択

`docs/blog-prompt-templates.md`から適切なテンプレートを選択

### 3. 具体的な情報を収集

- ReRank AIの実際のデータ（ユーザー統計、改善事例など）
- 競合記事の分析（差別化ポイントの特定）
- ターゲット読者の課題

### 4. プロンプトを生成

テンプレートに具体的な情報を埋め込んで、AIに渡すプロンプトを完成させる

### 5. 記事を生成

生成された記事をレビューし、必要に応じて修正・追加

## 執筆スタイルの重要原則

### 1. プロフェッショナルで人間らしいトーン
- ロボットっぽい表現、堅苦しい構造、不自然な流れを完全に除去

- 「この記事では」「本記事では」「まとめると」などのAIらしい表現を絶対に使わない

【必須要素】

- ReRank AIの実際のユーザーデータを含める（例：平均順位改善率、通知頻度など）

- ReRank AIのユーザー統計データ

4. 差別化ポイント
- 競合記事では触れられていない視点
- ReRank AI独自の機能やアプローチ
- 「併用する使い方」など、新しい視点

【記事構造】

6. 使い分け方（800-1,000文字）
7. 実際の改善事例（500-700文字）
8. まとめ（200-300文字）
   - 次のアクションを促す（CTA）
   - 関連記事へのリンク
9. 参考資料・出典（必須）
   - 記事末尾に「## 参考資料・出典」セクションを配置
   - 出典はID付きで記載（`<a id="ref1"></a>`形式）
10. 最終更新日
    - 記事末尾に「最終更新日：YYYY年MM月DD日」を記載

【執筆時の注意点】

### AIらしさを完全に排除する
以下の表現は絶対に使わないでください：
- ❌ 「この記事では」「本記事では」「まとめると」
- ❌ 「以上のように」「以下に示すように」
- ❌ 「まず」「次に」「最後に」などの機械的な接続詞の連続

代わりに：
- ✅ 読者の課題や疑問から自然に始める
- ✅ 具体例や事例から入る
- ✅ 会話調で温かく自然な感じで書く

## データ収集のヒント

### ReRank AIの実際のデータ

- ユーザー統計（平均順位改善率、通知頻度など）
- 改善事例（匿名化）
- よくある質問やフィードバック

### 競合記事の分析

- 主要なキーワードで検索
- 上位記事の内容を分析
- 不足している視点や深掘りポイントを特定

### ターゲット読者の課題

- ユーザーからの問い合わせ
- サポートチケットの分析
- コミュニティでの質問

## 記事の品質チェック

生成された記事について、以下を確認：

1. **独自性**: 一般的な情報ではなく、ReRank AIの独自の視点やデータが含まれているか
2. **深掘り**: 表面的な説明ではなく、具体的な実例やデータに基づいた深い分析があるか
3. **実用性**: 読者が実際に行動できる具体的なアドバイスがあるか
4. **差別化**: 競合記事との違いが明確か
5. **SEO要件**: 適切なキーワードが含まれ、メタディスクリプションが適切か
