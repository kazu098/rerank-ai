# GSC連携エラー

Google Search Console（GSC）との連携時にエラーが発生した場合の対処法を説明します。

## よくあるエラー

### 認証エラー（401 Unauthorized）

**原因**: Googleアカウントの認証トークンが期限切れ

**対処法**:
1. 設定ページの「GSC連携」を開く
2. 「再認証」をクリック
3. Googleアカウントで再認証

### 権限エラー（403 Forbidden）

**原因**: GSCへのアクセス権限がない、または再認証したアカウントがサイトの所有者/検証者として登録されていない

**対処法**:

#### 1. Google Search Consoleでサイトの所有者/検証者を確認

1. [Google Search Console](https://search.google.com/search-console)にアクセス
2. 左メニューから「設定」→「所有権の確認」を開く
3. 現在ログインしているGoogleアカウントが、サイトの所有者または検証者として登録されているか確認

#### 2. 再認証したアカウントがサイトの所有者/検証者として登録されていない場合

**問題**: 再認証したGoogleアカウントが、そのサイトの所有者/検証者として登録されていない場合、403エラーが発生します。

**解決方法**:

**方法A: サイトの所有者/検証者として登録する（推奨）**

1. [Google Search Console](https://search.google.com/search-console)にアクセス
2. 再認証したGoogleアカウントでログイン
3. 左メニューから「設定」→「所有権の確認」を開く
4. 「所有権を確認する方法を追加」をクリック
5. **HTMLタグ**の方法を選択（最も簡単）
6. 表示されたメタタグをコピー
7. サイトの`<head>`セクションにメタタグを追加
8. 「確認」をクリック

**方法B: 正しいGoogleアカウントで再認証する**

1. ReRank AIからログアウト
2. サイトの所有者/検証者として登録されているGoogleアカウントで再ログイン
3. 設定ページの「GSC連携」を開く
4. 「再認証」をクリック
5. 正しいGoogleアカウントで再認証

#### 3. 複数のGoogleアカウントで管理している場合

- **kazutaka.yoshinaga@gmail.com**で正常動作している場合、そのアカウントがサイトの所有者/検証者として登録されています
- 別のGmailアカウントで403エラーが発生する場合、そのアカウントがサイトの所有者/検証者として登録されていない可能性があります
- 解決方法: 上記の方法Aまたは方法Bを実行してください

#### 4. 再認証後も403エラーが発生する場合

再認証後も403エラーが発生する場合、以下の可能性があります：

1. **再認証したアカウントがサイトの所有者/検証者として登録されていない**
   - 上記の方法Aまたは方法Bを実行してください

2. **アクセストークンが更新されていない**
   - ダッシュボードにアクセスして、自動的にトークンが更新されるのを待つ
   - または、設定ページの「GSC連携」を開いて「再認証」をクリック

3. **サイトURLの形式が正しくない**
   - サイトURLが`https://example.com/`の形式で正しく登録されているか確認
   - 必要に応じて、サイトを削除して再登録してください

### トークン期限切れ

**原因**: リフレッシュトークンが期限切れ

**対処法**:
1. 設定ページの「GSC連携」を開く
2. 「再認証」をクリック
3. Googleアカウントで再認証
