# ReRank AIで記事を監視して通知を受け取る方法

分析が完了した記事を監視対象に追加することで、順位の変化を自動的に追跡できます。このガイドでは、記事を監視対象に追加し、ダッシュボードで確認して通知を設定する方法を詳しく解説します。

## この記事でわかること

- **記事の監視開始**: 分析結果画面から監視対象に追加する方法
- **ダッシュボードでの確認**: 監視中の記事の順位をダッシュボードで確認する方法
- **通知設定**: 順位下落を検知したら自動で通知を受け取る設定方法（アラート設定・メール・Slack）
- **Slackの通知先**: DMまたはチャンネルを選んで通知を受け取る方法

## 前提条件

このガイドを進める前に、以下の設定が完了している必要があります：

1. [アカウント作成](/ja/blog/getting-started#1-アカウント作成1分)が完了している
2. [Google Search Consoleプロパティの選択](/ja/blog/getting-started#2-google-search-consoleプロパティの選択1分)が完了している
3. [記事の分析開始](/ja/blog/getting-started#3-記事の分析開始1分)および[分析結果の確認](/ja/blog/getting-started#4-分析結果の確認1分)まで完了している

## 1. 記事を監視対象に追加（1分）

分析が完了した記事を監視対象に追加することで、順位の変化を自動的に追跡できます。

### 監視を開始する手順

1. **分析結果画面で「監視を開始」または「監視対象に追加」ボタンをクリック**

記事の分析が終わると[結果画面](/ja/blog/getting-started#4-分析結果の確認1分)が表示されます。結果画面にある「この記事を監視する」ボタンから、その記事を監視対象に追加できます。ここが監視の起点となります。

![監視を開始するボタン](/blog-images/monitoring-start-button.png)


2. **ダッシュボードの記事一覧で監視状況を確認**

監視を開始した記事は、ダッシュボードの記事一覧に表示されます。一覧では各記事ごとに監視のオン・オフを切り替えられます。

![記事一覧画面](/blog-images/monitoring-article-list.png)

3. **必要に応じて監視設定を確認**

監視するキーワードなどは、分析時に登録した内容がそのまま使われます。記事詳細や設定画面で必要に応じて確認できます。

![監視設定画面](/blog-images/monitoring-settings.png)

### 監視の仕組み

- **自動チェック**: 監視を開始した記事は、毎日自動的に順位をチェックされます
- **順位の追跡**: Google Search Consoleのデータを使用して、正確な順位を取得します
- **変化の検知**: 順位が下落した場合、自動的に検知されます

### よくある質問

**Q: 監視を開始するには、分析が完了している必要がありますか？**
A: はい、監視を開始するには、記事の分析が完了している必要があります。分析結果画面の「監視を開始」等のボタンから監視対象に追加できます。分析が完了していない場合は、「分析中」と表示されます。

**Q: 複数の記事を一度に監視対象に追加できますか？**
A: 現在は、1記事ずつ監視対象に追加する必要があります。今後、一括追加機能を追加予定です。

## 2. ダッシュボードで監視状況を確認（1分）

ダッシュボードでは、監視中の記事の順位や変化を確認できます。

### ダッシュボードの見方

1. ダッシュボードのトップページを開く

![ダッシュボードトップ](/blog-images/monitoring-dashboard-top.png)

2. 監視中の記事の詳細を確認

![記事詳細画面](/blog-images/monitoring-article-detail.png)

### ダッシュボードで確認できる情報

- **現在の順位**: 各キーワードの現在の順位
- **順位の推移**: 過去の順位の変化をグラフで表示
- **順位の変化**: 前回チェック時からの順位の変化（上昇/下落）
- **インプレッション数**: 各キーワードのインプレッション数
- **クリック数**: 各キーワードのクリック数

### よくある質問

**Q: 順位はどのくらいの頻度で更新されますか？**
A: 順位は毎日、日本時間の0時に自動的にチェックされます。Google Search Consoleのデータ更新タイミングにより、1-2日遅れる場合があります。

**Q: 過去の順位データ（分析結果）はどのくらい保存されますか？**
A: プランによって異なります。

| プラン | 分析結果の保存期間 |
|--------|-------------------|
| 無料プラン | 無制限 |
| スタータープラン | 30日間 |
| スタンダードプラン | 90日間 |
| ビジネスプラン | 無制限 |

## 3. 通知設定（2分）

順位が下落した際に、通知を受け取る設定を行います。通知を受け取ることで、迅速に対応できます。

### アラート設定（通知条件）

順位下落の通知条件は、**アラート設定**でまとめて変更できます。

- **ダッシュボードの記事一覧**にある「アラート設定」ボタンからモーダルを開くか、**設定メニューの「アラート設定」**ページで設定します。
- 設定できる主な項目：
  - **順位下落の閾値**: 何位下がったら通知するか（例: 2位）
  - **キーワード下落の閾値**: キーワードの順位が何位以下で通知するか（例: 10位）
  - **比較期間**: 何日前の順位と比較するか（例: 7日間）
  - **連続下落日数**・**最小インプレッション**・**通知クールダウン**など

![通知条件の設定画面](/blog-images/monitoring-notification-conditions.png)

### メール通知の設定

1. **設定ページ**の「通知設定」を開く
2. 「メール通知」を有効化し、通知を受け取るメールアドレスを確認

![通知設定画面](/blog-images/monitoring-notification-settings.png)

### Slack通知の設定（オプション）

チームで共有したい場合は、Slack通知も設定できます。

1. 設定ページの「**Slack連携**」でSlackアプリをインストール・連携する

![Slack連携画面](/blog-images/monitoring-slack-connect.png)

2. 連携後、**通知先（DMまたはチャンネル）を選択**する  
   - **自分のDM**: 通知を自分あてのDMに送る  
   - **チャンネル**: ワークスペース内のパブリック／プライベートチャンネルを一覧から選択し、そのチャンネルに通知を送る  
   設定ページの通知先ドロップダウンで、DMかチャンネルのいずれかを選べます。

![Slackチャンネル選択画面](/blog-images/monitoring-slack-channel.png)

### 通知の内容

通知には、以下の情報が含まれます：

- 順位が下がった記事のURL
- 順位が下がったキーワード
- 現在の順位と前回の順位
- 改善案の概要
- 詳細な分析結果へのリンク

### よくある質問

**Q: 通知はどのくらいの頻度で届きますか？**
A: 順位が設定した閾値を下回った場合、即座に通知が届きます。通常、1日1回、日本時間の0時に順位をチェックします。ただし、GSC APIのデータ更新タイミングにより、通知が1-2日遅れる場合があります。

**Q: メール通知とSlack通知の両方を設定できますか？**
A: はい、メール通知とSlack通知の両方を設定できます。両方の通知先に同じ内容が送信されます。

**Q: 通知を無効化できますか？**
A: はい、設定ページで通知を無効化できます。無効化しても、ダッシュボードで順位の変化を確認することはできます。

## まとめ

ReRank AIで記事を監視し、通知を受け取ることで、順位の変化を常に把握できます。このガイドに従って設定すれば、5分で監視と通知の設定が完了します。

**主なメリット**:
- 順位監視の自動化（手動チェックが不要）
- 順位下落の自動検知（即座に通知）
- ダッシュボードでの可視化（順位の推移をグラフで確認）
- 時間の大幅な削減（週5-10時間削減可能）

### 次のステップ

1. **監視を開始**: 分析結果画面の「監視を開始」等から記事を監視対象に追加
2. **通知を設定**: アラート設定・メール・Slack（DMまたはチャンネル）で通知を受け取る設定
3. **ダッシュボードを確認**: 定期的にダッシュボードで順位の変化を確認
4. **改善案を実装**: 通知を受け取ったら、改善案を実装して順位を回復

まずは1記事から監視を開始して、効果を実感してください。

## トラブルシューティング

### よくある問題と解決方法

#### 問題1: 監視を開始できない

**症状**: 分析結果画面に「監視を開始」等のボタンが表示されない、またはクリックしてもエラーが発生する

**解決方法**:
1. 記事の分析が完了しているか確認（「分析中」の場合は完了まで待つ）
2. プランの制限を確認（監視記事数の上限に達していないか）
3. ダッシュボードの記事一覧では、各記事の監視トグルでオン・オフを切り替えられる。そこで監視を有効にしてみる
4. ページをリロードして再試行し、ブラウザのコンソールでエラーの詳細を確認する

#### 問題2: 通知が届かない

**症状**: 順位が下がっているのに通知が届かない

**解決方法**:
1. 設定ページでメール通知またはSlack通知が有効か確認する
2. **アラート設定**で通知条件（順位下落の閾値・比較期間など）が適切か確認する
3. メールの迷惑メールフォルダを確認する
4. Slack通知の場合、**通知先（DMまたはチャンネル）**が設定ページで正しく選択されているか確認する
5. 順位チェックのタイミングを確認する（GSC APIのデータ更新を待つ）

#### 問題3: ダッシュボードに順位が表示されない

**症状**: 監視を開始したのに、ダッシュボードに順位が表示されない

**解決方法**:
1. 監視開始から24時間以上経過しているか確認（初回の順位チェックまで時間がかかる場合がある）
2. Google Search Consoleのデータ更新を待つ（通常、1-2日かかる）
3. 記事のURLが正しく登録されているか確認
4. GSC連携が正常に動作しているか確認

#### 問題4: 順位データが更新されない

**症状**: 順位が更新されない、古いデータが表示される

**解決方法**:
1. Google Search Consoleのデータ更新を待つ（通常、1-2日かかる）
2. 最後のチェック日時を確認
3. ページをリロードして、最新のデータを取得
4. GSC連携を再確認（認証が有効か確認）

## よくある質問（FAQ）

### Q1: 監視を開始すると、どのくらいの頻度で順位をチェックしますか？

A: 監視を開始した記事は、毎日、日本時間の0時に自動的に順位をチェックされます。Google Search Consoleのデータ更新タイミングにより、1-2日遅れる場合があります。

### Q2: 監視中の記事を削除できますか？

A: はい、監視中の記事を削除できます。記事の詳細ページから「監視を停止」をクリックすると、監視が停止されます。記事自体を削除することもできます。

### Q3: 通知の閾値はどのように設定すればいいですか？

A: 通知の閾値は、記事の重要度や目標順位に応じて設定してください。一般的には：
- **順位下落の閾値**: 2-5位（重要な記事の場合は1位）
- **キーワード下落の閾値**: 10-20位（重要なキーワードの場合は5位）
- **比較期間**: 7-14日前（週次で確認する場合は7日、月次で確認する場合は30日）

### Q4: 複数の通知先を設定できますか？

A: はい、メール通知とSlack通知の両方を有効にできます。両方有効にすると、同じ内容がメールとSlackの両方に送信されます。Slackの通知先は、**自分のDM**か**1つのチャンネル**のいずれかを選択します。設定ページの通知先で、DMまたはチャンネルを選べます。

### Q5: 通知の内容をカスタマイズできますか？

A: 現在は、通知の内容をカスタマイズすることはできません。今後、通知の内容をカスタマイズする機能を追加予定です。

### Q6: 監視中の記事の順位が上がった場合も通知を受け取れますか？

A: 現在は、順位下落の通知のみ対応しています。順位上昇の通知機能は、今後追加予定です。
