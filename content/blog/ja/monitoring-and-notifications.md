# ReRank AIで記事を監視して通知を受け取る方法

分析が完了した記事を監視対象に追加することで、順位の変化を自動的に追跡できます。このガイドでは、記事を監視対象に追加し、ダッシュボードで確認して通知を設定する方法を詳しく解説します。

## この記事でわかること

- **記事の監視開始**: 分析が完了した記事を監視対象に追加する方法
- **ダッシュボードでの確認**: 監視中の記事の順位をダッシュボードで確認する方法
- **通知設定**: 順位下落を検知したら自動で通知を受け取る設定方法
- **通知の種類**: メール通知とSlack通知の設定方法

## 前提条件

このガイドを進める前に、以下の設定が完了している必要があります：

1. [アカウント作成](/blog/getting-started#1-アカウント作成1分)が完了している
2. [Google Search Consoleとの連携](/blog/getting-started#2-google-search-consoleとの連携2分)が完了している
3. [記事の登録と分析](/blog/getting-started#3-記事の登録1分)が完了している

## 1. 記事を監視対象に追加（1分）

### 監視の仕組み

- **自動チェック**: 監視を開始した記事は、毎日自動的に順位をチェックされます
- **順位の追跡**: Google Search Consoleのデータを使用して、正確な順位を取得します
- **変化の検知**: 順位が下落した場合、自動的に検知されます

### よくある質問

**Q: 監視を開始するには、分析が完了している必要がありますか？**
A: はい、監視を開始するには、記事の分析が完了している必要があります。分析が完了していない場合は、「分析中」と表示されます。

**Q: 複数の記事を一度に監視対象に追加できますか？**
A: 現在は、1記事ずつ監視対象に追加する必要があります。今後、一括追加機能を追加予定です。

### ダッシュボードで確認できる情報

- **現在の順位**: 各キーワードの現在の順位
- **順位の推移**: 過去の順位の変化をグラフで表示
- **順位の変化**: 前回チェック時からの順位の変化（上昇/下落）
- **インプレッション数**: 各キーワードのインプレッション数
- **クリック数**: 各キーワードのクリック数

### よくある質問

**Q: 順位はどのくらいの頻度で更新されますか？**
A: 順位は毎日、日本時間の0時に自動的にチェックされます。Google Search Consoleのデータ更新タイミングにより、1-2日遅れる場合があります。

**Q: 過去の順位データはどのくらい保存されますか？**
A: プランによって異なります：
- 無料プラン・スタータープラン: 30日間
- スタンダードプラン: 90日間
- ビジネスプラン: 無制限

## 3. 通知設定（2分）

順位が下落した際に、通知を受け取る設定を行います。通知を受け取ることで、迅速に対応できます。

### メール通知の設定

1. 設定ページの「通知設定」を開く

![通知設定画面](blog-images/monitoring-notification-settings.png)
*画像の内容: 通知設定ページでメール通知の設定項目が表示されている画面*
*画像ファイル名: monitoring-notification-settings.png*

*画像のサイズ: 幅1200px、高さは自動*
*ハイライト: メール通知のトグルスイッチを赤い枠で囲む*

2. 「メール通知」を有効化
3. 通知を受け取るメールアドレスを確認
4. 通知条件を設定：
   - **順位下落の閾値**: 何位下がったら通知するか（デフォルト: 2位）
   - **キーワード下落の閾値**: キーワードの順位が何位下がったら通知するか（デフォルト: 10位）
   - **比較期間**: 何日前の順位と比較するか（デフォルト: 7日前）

### 通知の内容

通知には、以下の情報が含まれます：

- 順位が下がった記事のURL
- 順位が下がったキーワード
- 現在の順位と前回の順位
- 改善案の概要
- 詳細な分析結果へのリンク

### よくある質問

**Q: 通知はどのくらいの頻度で届きますか？**
A: 順位が設定した閾値を下回った場合、即座に通知が届きます。通常、1日1回、日本時間の0時に順位をチェックします。ただし、GSC APIのデータ更新タイミングにより、通知が1-2日遅れる場合があります。

**Q: メール通知とSlack通知の両方を設定できますか？**
A: はい、メール通知とSlack通知の両方を設定できます。両方の通知先に同じ内容が送信されます。

**Q: 通知を無効化できますか？**
A: はい、設定ページで通知を無効化できます。無効化しても、ダッシュボードで順位の変化を確認することはできます。

## まとめ

ReRank AIで記事を監視し、通知を受け取ることで、順位の変化を常に把握できます。このガイドに従って設定すれば、5分で監視と通知の設定が完了します。

**主なメリット**:
- 順位監視の自動化（手動チェックが不要）
- 順位下落の自動検知（即座に通知）
- ダッシュボードでの可視化（順位の推移をグラフで確認）
- 時間の大幅な削減（週5-10時間削減可能）

### 次のステップ

1. **監視を開始**: 分析が完了した記事を監視対象に追加
2. **通知を設定**: メールまたはSlackで通知を受け取る設定
3. **ダッシュボードを確認**: 定期的にダッシュボードで順位の変化を確認
4. **改善案を実装**: 通知を受け取ったら、改善案を実装して順位を回復

まずは1記事から監視を開始して、効果を実感してください。

## トラブルシューティング

### よくある問題と解決方法

#### 問題1: 監視を開始できない

**症状**: 「監視を開始」ボタンが表示されない、またはクリックしてもエラーが発生する

**解決方法**:
1. 記事の分析が完了しているか確認（「分析中」の場合は完了まで待つ）
2. プランの制限を確認（監視記事数の上限に達していないか）
3. ページをリロードして再試行
4. ブラウザのコンソールでエラーの詳細を確認

#### 問題2: 通知が届かない

**症状**: 順位が下がっているのに通知が届かない

**解決方法**:
1. 通知設定を確認（設定ページで有効になっているか）
2. 通知条件を確認（閾値が適切か）
3. メールの迷惑メールフォルダを確認
4. Slack通知の場合、チャンネルが正しく選択されているか確認
5. 順位チェックのタイミングを確認（GSC APIのデータ更新を待つ）

#### 問題3: ダッシュボードに順位が表示されない

**症状**: 監視を開始したのに、ダッシュボードに順位が表示されない

**解決方法**:
1. 監視開始から24時間以上経過しているか確認（初回の順位チェックまで時間がかかる場合がある）
2. Google Search Consoleのデータ更新を待つ（通常、1-2日かかる）
3. 記事のURLが正しく登録されているか確認
4. GSC連携が正常に動作しているか確認

#### 問題4: 順位データが更新されない

**症状**: 順位が更新されない、古いデータが表示される

**解決方法**:
1. Google Search Consoleのデータ更新を待つ（通常、1-2日かかる）
2. 最後のチェック日時を確認
3. ページをリロードして、最新のデータを取得
4. GSC連携を再確認（認証が有効か確認）

## よくある質問（FAQ）

### Q1: 監視を開始すると、どのくらいの頻度で順位をチェックしますか？

A: 監視を開始した記事は、毎日、日本時間の0時に自動的に順位をチェックされます。Google Search Consoleのデータ更新タイミングにより、1-2日遅れる場合があります。

### Q2: 監視中の記事を削除できますか？

A: はい、監視中の記事を削除できます。記事の詳細ページから「監視を停止」をクリックすると、監視が停止されます。記事自体を削除することもできます。

### Q3: 通知の閾値はどのように設定すればいいですか？

A: 通知の閾値は、記事の重要度や目標順位に応じて設定してください。一般的には：
- **順位下落の閾値**: 2-5位（重要な記事の場合は1位）
- **キーワード下落の閾値**: 10-20位（重要なキーワードの場合は5位）
- **比較期間**: 7-14日前（週次で確認する場合は7日、月次で確認する場合は30日）

### Q4: 複数の通知先を設定できますか？

A: はい、メール通知とSlack通知の両方を設定できます。また、Slack通知では複数のチャンネルに通知を送信することもできます。

### Q5: 通知の内容をカスタマイズできますか？

A: 現在は、通知の内容をカスタマイズすることはできません。今後、通知の内容をカスタマイズする機能を追加予定です。

### Q6: 監視中の記事の順位が上がった場合も通知を受け取れますか？

A: 現在は、順位下落の通知のみ対応しています。順位上昇の通知機能は、今後追加予定です。
