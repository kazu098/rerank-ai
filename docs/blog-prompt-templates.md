# ブログ記事生成プロンプトテンプレート

ReRank AIのブログ記事を生成するためのプロンプトテンプレート集。

## 基本方針

- **独自性**: 一般的な情報ではなく、ReRank AIの独自の視点やデータを含める
- **深掘り**: 表面的な説明ではなく、具体的な実例やデータに基づいた深い分析
- **実用性**: 読者が実際に行動できる具体的なアドバイス
- **差別化**: 競合記事との違いを明確にする

---

## テンプレート1: 比較記事（競合ツールとの比較）

```
# 記事タイトル: [ツール名] vs ReRank AI：[具体的な比較ポイント]

## 記事の目的
- [ツール名]ユーザーにReRank AIの価値を伝える
- 両ツールの違いを明確にし、使い分け方を提示
- ReRank AIの独自の強みを強調

## 必須要素

### 1. 独自の視点
- ReRank AIの実際のユーザーデータや事例を含める
- 「一般的には○○と言われているが、実際のデータでは...」という独自の分析
- ReRank AIを使った具体的な改善事例（匿名化）

### 2. 深掘りポイント
- 機能比較だけでなく、「なぜその違いが重要なのか」を説明
- 実際の使用シーンでの使い分けを具体的に提示
- コストパフォーマンスの詳細な分析（ROI計算など）

### 3. 実データ・実例
- ReRank AIのユーザー統計データ（例：平均順位改善率、通知頻度など）
- 実際の改善事例（順位が○位から○位に改善、など）
- コスト比較の具体的な数値

### 4. 差別化ポイント
- 競合記事では触れられていない視点
- ReRank AI独自の機能やアプローチ
- 「併用する使い方」など、新しい視点

## 記事構造

1. **導入（200-300文字）**
   - 読者の課題を明確に
   - この記事で解決できることを提示

2. **[ツール名]とは（500-700文字）**
   - 基本的な機能説明
   - 強み・弱みの分析
   - 一般的な評価（客観的に）

3. **ReRank AIとは（500-700文字）**
   - 基本的な機能説明
   - 独自の強みを強調
   - 実際のユーザーデータを含める

4. **詳細比較（1,500-2,000文字）**
   - 機能別の比較表
   - 各機能の違いを深掘り
   - 「なぜその違いが重要なのか」を説明
   - 実際の使用シーンでの比較

5. **使い分け方（800-1,000文字）**
   - 具体的な使用シーンを提示
   - 併用する使い方の提案
   - コストパフォーマンスの分析

6. **実際の改善事例（500-700文字）**
   - ReRank AIを使った具体的な改善事例
   - 数値データを含める
   - 匿名化したユーザーの声

7. **まとめ（200-300文字）**
   - 要点の整理
   - 次のアクションを提示

## 執筆時の注意点

- 競合ツールを貶すのではなく、客観的に比較する
- ReRank AIの強みを自然に強調
- 読者が実際に行動できる具体的なアドバイスを含める
- 専門用語は説明を入れる
- データや統計は出典を明記（可能な場合）
```

## テンプレート2: SEO教育コンテンツ

```
# 記事タイトル: [SEOトピック]：[具体的な課題と解決策]

## 記事の目的
- SEOに関する知識を深める
- ReRank AIの価値を間接的に伝える
- 実践的なアドバイスを提供

## 必須要素

### 1. 独自の視点
- ReRank AIが分析した実際のデータを含める
- 「一般的なSEO対策では見落としがちなポイント」を提示
- 競合分析の重要性を具体的なデータで説明

### 2. 深掘りポイント
- 「なぜその対策が重要なのか」をデータで説明
- 実際の失敗事例と成功事例を比較
- 対策の優先順位を明確に提示

### 3. 実データ・実例
- ReRank AIの分析データ（例：順位下落の主な原因、改善効果など）
- 実際のサイトの改善事例
- 統計データや調査結果の引用

### 4. 実用性
- 読者が今すぐ実行できる具体的な手順
- チェックリストやテンプレート
- ツールの使い方（ReRank AIを含む）

## 記事構造

1. **導入（200-300文字）**
   - 読者が直面している課題を提示
   - この記事で解決できることを明確に

2. **問題の理解（800-1,000文字）**
   - 課題の背景を説明
   - なぜその課題が重要なのかをデータで説明
   - ReRank AIの分析データを含める

3. **原因の分析（1,000-1,500文字）**
   - 主な原因を詳しく解説
   - 実際のデータや事例を含める
   - 競合分析の重要性を説明

4. **解決策（1,500-2,000文字）**
   - 具体的な解決策を提示
   - ステップバイステップの手順
   - ReRank AIを使った実践方法

5. **実例・ケーススタディ（800-1,000文字）**
   - 実際の改善事例
   - 数値データを含める
   - 改善前後の比較

6. **まとめ・次のステップ（300-400文字）**
   - 要点の整理
   - 読者が取るべき次のアクション

## 執筆時の注意点

- SEOの基本から説明する（初心者にも理解できる）
- 専門用語は説明を入れる
- データや統計は出典を明記
- ReRank AIの価値を自然に伝える（押し付けがましくない）
```

## テンプレート3: 使い方ガイド

```
# 記事タイトル: [機能名]の使い方：[具体的な目的]

## 記事の目的
- ReRank AIの機能の使い方を詳しく説明
- ユーザーのオンボーディングを支援
- 検索流入を獲得（「ReRank AI 使い方」など）

## 必須要素

### 1. スクリーンショット
- 各ステップで実際の画面キャプチャを含める
- 重要な部分をハイライト
- 番号や矢印で手順を明確に

### 2. 深掘りポイント
- 「なぜその設定が重要なのか」を説明
- よくある間違いとその対処法
- 応用的な使い方も紹介

### 3. 実例
- 実際の使用シーンを想定した例
- 設定例や入力例
- 期待される結果

### 4. トラブルシューティング
- よくある問題と解決方法
- エラーメッセージの対処法
- サポートへの連絡方法

## 記事構造

1. **導入（200-300文字）**
   - この機能でできることを説明
   - 読者の課題を明確に

2. **前提条件（200-300文字）**
   - 必要な準備や設定
   - 前提となる知識

3. **基本的な使い方（1,500-2,000文字）**
   - ステップバイステップの手順
   - 各ステップのスクリーンショット
   - 重要なポイントの説明

4. **応用的な使い方（800-1,000文字）**
   - より高度な設定や使い方
   - 効率的な使い方のコツ
   - 実際の使用シーン

5. **よくある質問（500-700文字）**
   - よくある問題と解決方法
   - エラーの対処法

6. **まとめ（200-300文字）**
   - 要点の整理
   - 次のステップ

## 執筆時の注意点

- 初心者にも理解できるように説明
- 専門用語は説明を入れる
- スクリーンショットは最新の画面を使用
- 実際のユーザーの声やフィードバックを含める（可能な場合）
```

## プロンプト生成のワークフロー

### ステップ1: 記事の種類を選択
- 比較記事
- SEO教育コンテンツ
- 使い方ガイド
- その他

### ステップ2: テンプレートを選択
- 上記のテンプレートから適切なものを選択

### ステプ3: 具体的な情報を入力
- ターゲット読者
- 記事の目的
- 含めるべきデータや事例
- 差別化ポイント

### ステップ4: プロンプトを生成
- テンプレートに具体的な情報を埋め込む
- AIに渡すプロンプトを完成させる

### ステップ5: 記事を生成・レビュー
- AIで記事を生成
- 独自性と深掘りを確認
- 必要に応じて修正・追加

---

## チェックリスト

記事を公開する前に、以下を確認：

- [ ] 独自の視点やデータが含まれているか
- [ ] 表面的な説明ではなく、深掘りされているか
- [ ] 実データや実例が含まれているか
- [ ] 読者が実際に行動できる具体的なアドバイスがあるか
- [ ] 競合記事との差別化ができているか
- [ ] SEO要件（キーワード、メタディスクリプションなど）を満たしているか
- [ ] スクリーンショットや画像が適切に配置されているか
- [ ] 専門用語に説明が入っているか
- [ ] データや統計の出典が明記されているか（可能な場合）

## 使用例

### 例1: Ahrefs比較記事を生成する場合

```
プロンプト:
上記の「テンプレート1: 比較記事」を使用して、
「Ahrefs vs ReRank AI：違いと使い分け方」という記事を生成してください。

具体的な要件:
- ターゲット: Ahrefsを使用しているSEO担当者
- 独自データ: ReRank AIのユーザー統計（平均順位改善率、通知頻度など）
- 深掘りポイント: コストパフォーマンスの詳細分析、実際の使用シーンでの比較
- 差別化: 「併用する使い方」という新しい視点
- 実例: ReRank AIを使った具体的な改善事例（匿名化）
```

### 例2: SEO教育コンテンツを生成する場合

```
プロンプト:
上記の「テンプレート2: SEO教育コンテンツ」を使用して、
「順位下落の原因と対策：競合分析の重要性」という記事を生成してください。

具体的な要件:
- ターゲット: 順位下落に悩むサイト運営者
- 独自データ: ReRank AIが分析した順位下落の主な原因データ
- 深掘りポイント: 競合分析の重要性を具体的なデータで説明、対策の優先順位
- 実例: ReRank AIを使った実際の改善事例
- 実用性: 今すぐ実行できる具体的な手順
```
