# ブログ記事生成プロンプトテンプレート

ReRank AIのブログ記事を生成するためのプロンプトテンプレート集。

## 基本方針

- **独自性**: 一般的な情報ではなく、ReRank AIの独自の視点やデータを含める
- **深掘り**: 表面的な説明ではなく、具体的な実例やデータに基づいた深い分析
- **実用性**: 読者が実際に行動できる具体的なアドバイス
- **差別化**: 競合記事との違いを明確にする
- **人間らしさ**: AIが書いたとわからない自然で読みやすい文章
- **信頼性**: 煽る表現を避け、客観的で信頼性の高いトーン

## テンプレート1: 比較記事（競合ツールとの比較）

```
# 記事タイトル: [ツール名] vs ReRank AI：[具体的な比較ポイント]

## 記事の目的
- [ツール名]ユーザーにReRank AIの価値を伝える
- 両ツールの違いを明確にし、使い分け方を提示
- ReRank AIの独自の強みを強調

## 必須要素

### 1. 独自の視点
- ReRank AIの実際のユーザーデータや事例を含める
- 「一般的には○○と言われているが、実際のデータでは...」という独自の分析
- ReRank AIを使った具体的な改善事例（匿名化）

### 2. 深掘りポイント
- 機能比較だけでなく、「なぜその違いが重要なのか」を説明
- 実際の使用シーンでの使い分けを具体的に提示
- コストパフォーマンスの詳細な分析（ROI計算など）

### 3. 実データ・実例
- ReRank AIのユーザー統計データ（例：平均順位改善率、通知頻度など）
- 実際の改善事例（順位が○位から○位に改善、など）
- コスト比較の具体的な数値

### 4. 差別化ポイント
- 競合記事では触れられていない視点
- ReRank AI独自の機能やアプローチ
- 「併用する使い方」など、新しい視点

## 記事構造

1. **導入（200-300文字）**
   - 読者の課題や疑問を提示
   - 記事で解決する内容を明示
   - 読むメリットを提示

2. **この記事でわかること（必須）**
   - 記事冒頭に「## この記事でわかること」セクションを配置
   - 主要ポイントを箇条書きで明確に提示（3-5項目）
   - LLMが引用しやすい構成にするため、最初に明確な回答を提示
   - 例：
     ```
     ## この記事でわかること
     
     - **[ツール名]の特徴**: 基本的な機能と強み・弱み
     - **ReRank AIの特徴**: 独自の強みと実際のユーザーデータ
     - **詳細比較**: 機能別の違いと「なぜその違いが重要なのか」
     - **使い分け方**: 具体的な使用シーンと併用の提案
     - **改善事例**: ReRank AIを使った具体的な改善事例と数値データ
     ```

3. **[ツール名]とは（500-700文字）**
   - 基本的な機能説明
   - 強み・弱みの分析
   - 一般的な評価（客観的に）
   - 出典を本文中に`[[1]](#ref1)`形式でインライン引用

4. **ReRank AIとは（500-700文字）**
   - 基本的な機能説明
   - 独自の強みを強調
   - 実際のユーザーデータを含める
   - 出典を本文中に`[[2]](#ref2)`形式でインライン引用

5. **詳細比較（1,500-2,000文字）**
   - 機能別の比較表
   - 各機能の違いを深掘り
   - 「なぜその違いが重要なのか」を説明
   - 実際の使用シーンでの比較
   - 重要なデータや統計は出典を明記

6. **使い分け方（800-1,000文字）**
   - 具体的な使用シーンを提示
   - 併用する使い方の提案
   - コストパフォーマンスの分析

7. **実際の改善事例（500-700文字）**
   - ReRank AIを使った具体的な改善事例
   - 数値データを含める
   - 匿名化したユーザーの声

8. **まとめ（200-300文字）**
   - 要点の整理
   - 次のアクションを促す（CTA）
   - 関連記事へのリンク

9. **参考資料・出典（必須）**
   - 記事末尾に「## 参考資料・出典」セクションを配置
   - 出典はID付きで記載（`<a id="ref1"></a>`形式）
   - 例：
     ```
     ## 参考資料・出典
     
     <a id="ref1"></a>
     **[1] タイトル**  
     出典元. URL
     ```

10. **最終更新日**
    - 記事末尾に「最終更新日：YYYY年MM月DD日」を記載

## 執筆時の注意点

### AIらしさを完全に排除する

以下の表現は絶対に使わないでください：
- ❌ 「この記事では」「本記事では」「まとめると」
- ❌ 「以上のように」「以下に示すように」
- ❌ 「まず」「次に」「最後に」などの機械的な接続詞の連続
- ❌ 「〜について解説します」「〜をご紹介します」などの形式的な導入

代わりに：
- ✅ 読者の課題や疑問から自然に始める
- ✅ 具体例や事例から入る
- ✅ 会話調で温かく自然な感じで書く
- ✅ 知識豊富な人間がカジュアルに説明しているようなトーン
- ✅ 文の長さを変化させ、リズムを作る
- ✅ 反復的または予測可能な表現を削除

### 煽る表現を避ける（信頼性重視）

以下のような強い表現は避け、客観的で信頼性の高いトーンを保つ：
- ❌ 「決定的な違い」「致命的な弱点」「危険なのか」などの煽る表現
- ❌ 「大きな違い」「大きく異なる」などの過度な強調
- ❌ 「単なる機能の違いではありません」などの断定的な表現

代わりに：
- ✅ 「違い」「主な違い」など、客観的な表現
- ✅ 「特有の課題」「制約」など、中立的な表現
- ✅ 「設計思想と用途に違いがあります」など、説明調の表現
- ✅ 事実に基づいた客観的な説明を心がける

### その他の注意点

- 競合ツールを貶すのではなく、客観的に比較する
- ReRank AIの強みを自然に強調
- 読者が実際に行動できる具体的なアドバイスを含める
- 専門用語は説明を入れる
- データや統計は必ず出典を明記
- **本文中にインライン引用**：`[[1]](#ref1)`形式で該当テキストに直接リンク
- **記事末尾に出典詳細**：`<a id="ref1"></a>`形式でID付きで記載
- 短い段落で構成（3-5文程度）
- 箇条書きや表を活用
- 重要なポイントは`<div class="important-point">`で囲んで強調
```

## プロンプト生成のワークフロー

### ステップ1: 記事の種類を選択
- 比較記事
- SEO教育コンテンツ
- 使い方ガイド
- その他

### ステップ2: テンプレートを選択
- 上記のテンプレートから適切なものを選択

### ステプ3: 具体的な情報を入力
- ターゲット読者
- 記事の目的
- 含めるべきデータや事例
- 差別化ポイント

### ステップ4: プロンプトを生成
- テンプレートに具体的な情報を埋め込む
- AIに渡すプロンプトを完成させる
- システムプロンプト（執筆スタイルの指示）を含める

### ステップ5: 記事を生成・レビュー
- AIで記事を生成
- 独自性と深掘りを確認
- AIらしい表現がないか確認
- 煽る表現がないか確認
- 必要に応じて修正・追加

### 執筆スタイル
- [ ] AIらしい表現（「この記事では」「まとめると」など）が使われていないか
- [ ] 煽る表現が使われていないか
- [ ] 自然で読みやすい文章になっているか
- [ ] 短い段落で構成されているか（3-5文程度）

### SEO・技術要件
- [ ] SEO要件（キーワード、メタディスクリプションなど）を満たしているか
- [ ] スクリーンショットや画像が適切に配置されているか
- [ ] 専門用語に説明が入っているか
- [ ] データや統計の出典が明記されているか
- [ ] 本文中にインライン引用（`[[1]](#ref1)`形式）が含まれているか
- [ ] 記事末尾に出典詳細（`<a id="ref1"></a>`形式）が含まれているか

## 使用例

### 例1: Ahrefs比較記事を生成する場合

```
システムプロンプト:
あなたは経験豊富なマーケティングライターです。SEO対策を意識しながら、読者に価値のある記事を執筆します。AIが書いたとわからないよう、自然で読みやすい文章を心がけます。

## 執筆スタイルの重要原則

### 1. プロフェッショナルで人間らしいトーン
- ロボットっぽい表現、堅苦しい構造、不自然な流れを完全に除去
- 会話調で温かく自然な感じで、プロフェッショナルだけど親しみやすいトーンを保つ
- 「この記事では」「本記事では」「まとめると」などのAIらしい表現を絶対に使わない

### 2. 自然な文章のリズムと流れ
- 文の長さを変化させ、ペースを調整
- 反復的または予測可能な表現を削除
- 微妙な人間らしい感情、スムーズな流れ、自然な強調を加える

### 3. 煽る表現を避ける（信頼性重視）
- 「決定的な違い」「致命的な弱点」などの煽る表現を避ける
- 「違い」「主な違い」など、客観的な表現を使用
- 事実に基づいた客観的な説明を心がける

ユーザープロンプト:
上記の「テンプレート1: 比較記事」を使用して、
「Ahrefs vs ReRank AI：違いと使い分け方」という記事を生成してください。

具体的な要件:
- ターゲット: Ahrefsを使用しているSEO担当者
- 独自データ: ReRank AIのユーザー統計（平均順位改善率、通知頻度など）
- 深掘りポイント: コストパフォーマンスの詳細分析、実際の使用シーンでの比較
- 差別化: 「併用する使い方」という新しい視点
- 実例: ReRank AIを使った具体的な改善事例（匿名化）

必須要素:
- 「この記事でわかること」セクションを冒頭に配置
- 本文中に出典をインライン引用（`[[1]](#ref1)`形式）
- 記事末尾に「参考資料・出典」セクションを配置
- 最終更新日を記載
```
