# デリバリー設計に基づく「やるべきこと」一覧

デリバリーファネル・3つの問い・勝ちパターンに沿って、**プロダクト/LP以外**のマーケ・ターゲティング・全体戦略でやるべきことを整理する。

---

## 1. デリバリーファネル：各段の「信じてもらうこと」とやるべきこと

| 段階 | 信じてもらうこと | 現状 | やるべきこと（マーケ・設計） |
|------|------------------|------|------------------------------|
| **獲得** | これ、自分の問題だ | 広告クリックはあるが「自分ごと」まで届いていない可能性 | ・**ターゲットを超特定**：ブログ/メディア運営・SEO担当・コンテンツ担当に絞る。広告オーディエンスをここに合わせる。<br>・**刺さる一言で出会いを設計**：`lp-messaging.md` の10個を広告・ブログ・SNSで統一して使う。「順位が下がってからじゃ遅い」など**問題を自分事にする一言**を前面に。 |
| **起動** | やればすぐ得する | 未ログインで順位＋ヒント1件は実装済み。Google連携で離脱あり | ・**初回体験の編集**：登録前の「順位確認」を主導線に（済）。連携離脱対策として「許可がある場合のみ」などの安心文言（済）。<br>・**Time to Value の計測**：未ログインで1回分析した人のうち何%が登録したか（Activation）を追う。 |
| **習慣** | これがないと戻れない | まだ計測・設計が薄い | ・**戻ってくるトリガー**：通知（順位下落・メンテリマインド）を「仕事の流れ」に乗せる。締切・定期レポート・Slack連携など。<br>・**価値が見えるタイミング**：分析結果・通知の中に「前回比」「改善案の採用結果」など、価値が積み上がって見える要素を入れる。 |
| **拡散** | 人に教えると自分の得になる | 紹介導線が弱い | ・**成果物の持ち出し**：分析レポート・改善案をエクスポート/共有するとき、ロゴ・「ReRank AIで分析」リンクを載せるオプション。共有前提のフォーマット（PDF/Notion風）を用意。<br>・紹介プログラム（無料分析回数など）は後追いでよい。まず「自然に露出する形」を設計。 |
| **収益** | 払う方が合理的 | 無料で十分と思われないようにする | ・**支払い理由の言語化**：時間短縮・精度・社内展開・監査ログなど、有料化の「理由」を事前に決め、LP・FAQ・営業資料に書く。<br>・無料枠を「試す用」に限定し、本格利用は有料であることを体験の流れで示す。 |

## 2. 3つの問い：ReRank AI の位置とやるべきこと

| 問い | 現状 | やるべきこと |
|------|------|--------------|
| **①既存の流れに乗れるか** | 広告は流れの一つ。SEO・GSC連携は「既存ツールの流れ」に乗っている。 | ・**チャネルを広げる**：広告だけにしない。SEO（「順位下落 原因」「SEO 記事 メンテ」などの記事）、GSC/ブログツール連携、パートナー（SEOコンサル・制作会社）のメニューに入れる。<br>・最初は「流れに寄せる」。ゼロからトラフィックを作りに行かず、検索・連携・紹介のどれか一点に絞る。 |
| **②最初の成果物が自動で出るか** | 未ログインでURL＋キーワード → 順位＋ヒント1件は実装済み。 | ・**入力の最小化**：キーワードはカンマ区切りで複数対応済み。必要なら「URLだけ入力でキーワード推測」を追加。<br>・**出力の最大化**：改善ヒント1件を「具体的な一文」にし、驚きが伝わるようにする。 |
| **③一人が連れてくる構造があるか** | 紹介機能は弱い。成果物の共有時に露出する形は未整備。 | ・**共有時にロゴ/リンクが載る**：分析結果のエクスポート・共有テンプレに「Powered by ReRank AI」や短いリンクを載せるオプションを用意。<br>・紹介コードは後からでよい。まず「成果物を外に出す＝自然に露出」を設計する。 |

## 3. マーケ・ターゲティング戦略でやるべきこと

### ターゲティング

- **超特定**：「SEO記事を書いたあと、メンテまで手が回らず、気づいたら順位が下がっていた人」に絞る。
- **広告**：オーディエンスをブログ運営・メディア運営・SEO・コンテンツマーケに限定。日本語は「問題を自分事にする」コピー＋リール動画で単価改善を狙う。
- **一言の統一**：LP・広告・ブログ・SNSで `lp-messaging.md` の刺さる一言から選び、**毎回同じ世界観**を積み上げる。バズより「刺さるテーマの連打」。

### チャネル（流れに乗る）

- **コンテンツ起点**：ブログで「順位下落 原因」「SEO 記事 メンテ」「GSC 活用」などを扱い、検索流入を取る。具体例・失敗談・比較・チェックリストが効く。
- **ワークフロー寄生**：GSC・Notion・Slackなど、既に使っているものに「連携するだけ」で入り込む。**導入摩擦の低さ**を訴求（1分で価値が出る）。
- **パートナー同居**：SEOコンサル・制作会社・BPOなどのメニューに組み込んでもらう。プロダクト単体より「パートナーが売りやすい形」に寄せる検討。

最初は**一点突破**。A/B/Cのうち、自分の強みに合う一つを選び、他は後回しでよい。

### 指標（起業初期で外さないもの）

- **Activation**：未ログインで1回分析した人のうち、アカウント作成した人の割合。
- **Time to Value**：URL入力から結果表示までの時間（短いほどよい）。
- **継続が起きる理由**：週1回でも「毎回同じ成果」が出ているか。通知・締切・定期レポートなど、戻ってくるトリガーが働いているか。

## 4. デリバリー設計テンプレに沿ったやるべきこと

- **一枚絵**：「出会い→初回→習慣→拡散→課金」を並べ、各段で「一言」「証拠」「体験」「指標」を書く。`delivery-analysis-fb-ad.md` と本ドキュメントで半分以上は埋まっている。残りは習慣・拡散・課金の「証拠」と「体験」を具体化する。
- **初回体験**：「入力→出力→驚き→次の一手」。入力は最小、出力は最大、驚きは比較/可視化、次の一手はボタンで提示。ここを作り切ると広告・営業の効きが良くなる。
- **セールス最小セット**：①1枚の価値提案、②デモ動画、③導入手順、④料金の考え方、⑤FAQ（「本当に使えるの？」「情報漏洩は？」を最初から潰す）。

## 5. 落とし穴を避けるために

- デリバリー＝広告運用ではない。**プロダクトが整ってから**広告を増やす。まずは起動（Activation・TtV）を固める。
- デリバリー＝SNSで頑張ることではない。**再現性のある仕組み**（チャネル・一言・指標）を決める。
- ターゲットを広げすぎない。**超特定のユーザー**で勝てる形にし、そこから広げる。
- プロダクトとデリバリーを分けない。届ける過程で分かった「刺さる言葉」「離脱ポイント」「導入摩擦」を**毎週プロダクトに反映**する。

## 6. 優先順位の提案

1. **今すぐ**：ターゲティングの絞り込み（広告オーディエンス）と、広告・LPの「一言」を `lp-messaging.md` に統一する。
2. **今すぐ〜短期（優先度高）**：**GSCで既にインプレッションが出ている競合・比較クエリを狙い撃ち**する。
   - **クエリ例**（Search Console実績）：`ahrefs 高い` `ahrefs比較` `semrush比較` `seo ahrefs vs` など。すでに検索結果に現れている＝「既存の流れ」に乗れる証拠。
   - **記事**：Ahrefs/SEMrush/Surferなどとの比較記事・価格比較・「〇〇が高い人向けの代替」を、`marketing-strategy.md` の比較記事テーマに沿って作成。順位を上げればクリックを取りにいける。
   - **広告**：同じキーワード（ahrefs 比較、semrush 比較、SEOツール 価格 など）で検索連動型やディスプレイのターゲティングをすると、**比較・選定中の層**に届きやすい。LPの「一言」は比較・価格ニーズに合わせる。
3. **短期**：Activation・Time to Value の計測を入れ、未ログイン→登録の転換を追う。習慣の「戻ってくるトリガー」（通知・Slack）を設計する。
4. **中期**：コンテンツ（SEO）かワークフロー寄生（連携訴求）かパートナーか、**一点**選んで流れに乗る。成果物の共有時にロゴ/リンクが載る形を用意する。
5. **その後**：支払い理由の言語化と、収益化の前に「払う方が合理的」と思わせる体験を設計する。

プロダクト/LPの改善に加え、**マーケ・ターゲティング・指標・チャネル**をデリバリーの「信念」に合わせて揃えると、届ける構造が一気に整う。
