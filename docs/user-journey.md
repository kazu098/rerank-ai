# ユーザージャーニー

## First Wow Experience の設計

「登録前に驚きを与える」ことが、ユーザー獲得の鍵です。

## ステップ1: 完全無料（未ログイン・GSC連携不要）

### 体験フロー

1. **ランディングページにアクセス**
   - 「URLを入力するだけで、改善案が届く」というメッセージ
   - 「デモを見る」ボタンも配置（成功事例を先に見せる）

2. **URL入力（設定ゼロ）**
   - ユーザーが自分の記事URLを入力
   - キーワードを入力（またはAIが自動推測）
   - **GSC連携は不要** - Serper APIで現在の順位を自動特定
   - 「分析開始」ボタンをクリック

3. **即座に結果表示（30秒以内）**
   - **「あなたの記事は現在○位です」**と現在の順位を表示
   - **「改善のヒント（見出し1つ分）」**を即座に表示
   - 例：「あなたの記事には『価格比較表』が欠けています。上位サイトは全員この要素を含んでいます。」

4. **誘導**
   - 「続き（具体的なリライト原稿）を見たい場合はアカウント作成」へ誘導
   - 「無料で3記事まで分析可能」というメッセージ

### 目的

- まず驚きを与え、価値を実感してもらう
- 登録のハードルを下げる（「まず試してみる」体験）

## ステップ2: 無料プラン（要アカウント作成）

### 体験フロー

1. **アカウント作成**
   - メールアドレスまたはGoogleアカウントで登録
   - 簡単なプロフィール入力（任意）

2. **完全な分析結果を閲覧**
   - 先ほどのURLの完全なリライト原稿を表示
   - コピー＆ペースト可能な形式

3. **GSC連携（任意・1クリック認証）**
   - 「Googleでログイン」ボタンをクリックするだけ（OAuth 2.0）
   - APIキーの発行などは不要
   - 順位下落時の自動通知を設定可能

4. **無料枠の利用**
   - 毎月3記事まで無料でスキャン可能
   - 順位下落時の通知メールを1件まで登録可能

### 目的

- 「自分で直すより圧倒的に速い」ことを実感させる
- 継続利用への動機付け

## ステップ3: 有料プランへの移行

### トリガー

- 無料枠（3記事）を使い切った時
- 順位下落通知を受け取って「もっと分析したい」と思った時
- 複数サイトを管理している時

### 移行の障壁を下げる施策

- 14日間の無料トライアル（有料プラン）
- クレジットカード登録不要でトライアル開始
- プラン比較表で明確に価値を提示

## 通知体験の設計

### 順位下落時の通知（自動）

**メッセージ例**:
```
【ReRank AI】異変を検知しました

あなたの記事「ポケトモ レビュー」が、12/11の4.1位から12/22の6.7位に下落しました（2.6位下落）。

原因分析:
「価格」「月額」などのキーワードが10位前後に転落し、全体の順位を下げています。
これらのキーワードは「検討末期」のユーザーが検索する重要なキーワードです。

改善案:
以下の項目が競合と比べて抜けているため、追加してください。

- 「価格比較表」が欠けている → 価格比較表を追加してください
- 「デメリット解消法」が欠けている → デメリット解消法のセクションを追加してください
- 「選定基準の比較」が欠けている → 選定基準の比較表を追加してください

詳細はこちら: [ダッシュボードへのリンク]
```

### 現状低迷時の通知（手動スキャン後）

**メッセージ例**:
```
【ReRank AI】伸びしろを発見しました

あなたの記事「〇〇の選び方」は現在20位ですが、1位の記事にある「選定基準の比較表」を取り入れるだけで、上位に食い込めます。

改善案:
以下の項目が上位サイトに共通して存在し、自社記事に欠けているため、追加してください。

- 「選定基準の比較表」が欠けている → 選定基準の比較表を追加してください
- 「実際の使用例」が欠けている → 実際の使用例のセクションを追加してください
- 「よくある質問（FAQ）」が欠けている → FAQセクションを追加してください

詳細はこちら: [ダッシュボードへのリンク]
```

## UI/UXの設計原則

### 1. 敷居の低さ

- ログイン前でも価値を体験できる
- URL入力だけで開始できる

### 2. 具体的な解決策

- 抽象的なアドバイスは一切排除
- コピー＆ペースト可能な形式

### 3. プッシュ型の体験

- ユーザーが「わざわざ見に行かなくていい」
- 通知で完結する体験

### 4. 状況別のメッセージング

- **下落時**: 警告（赤/黄のバッジ）で安心感を与える
- **低迷時**: チャンス（紫/緑のバッジ）で期待感を与える
